# Нейросети как обезболивающее

За последние тридцать дней на Инфостарте вышло больше десятка статей об интеграции LLM в 1С. LangGraph, ReAct-паттерн, Qwen, Claude, GigaChat, Gemini — нейросетевое бинго заполнено до краёв. Статья про AI-агента, который через LangGraph задаёт вопросы к 1С:ЗУП, собрала 78 комментариев. Тест языковых моделей для задач 1С — ещё 68. OneBase, open-source альтернатива платформе на Go, — рекордные 158 комментариев, хотя сообщество встретило её скептически.

И среди всего этого — статья «Зачем использовать docker?» с 65 комментариями.

Я перечитал этот тред несколько раз. Там люди в 2026 году обсуждают основы контейнеризации. Не Kubernetes, не Helm-чарты — просто Docker. И это не новички. Это опытные 1С-разработчики, которые genuinely не видели практического повода его использовать до сих пор. И именно это наблюдение открыло мне всю картину.

В треде про LangGraph-агента один из участников, starik-2005, задаёт закономерный вопрос: «Зачем Gemma-4:31b, когда есть Qwen 3.6-35B и GLM 4.7?» Разворачивается длинный спор о VRAM, RTX Blackwell, LoRA/DoRA fine-tuning. Умный разговор умных людей — но я замечаю: ни один не говорит «мы решили эту задачу встроенными средствами платформы». Потому что встроенных средств нет. Весь этот инженерный труд — сорок-шестьдесят часов настройки, как сам автор оценивает свой MVP, — направлен на то, чтобы бизнес-аналитик мог спросить: «В чём разница между начисленной суммой и удержаниями?» Вопрос, с которым нормальный BI-инструмент справился бы за секунды.

Самое красноречивое свидетельство — статья про Qwen Code CLI с подзаголовком «BSL Language Server + CodePilot1C MCP». Двадцать лет существования платформы — и только сейчас, в 2026 году, появляется нормальный language server для BSL. Не от 1С. От сообщества. С бесплатными двумя тысячами запросов в день и без VPN. Автор подчёркивает: это то, что у Java или TypeScript есть из коробки — умное автодополнение, проверка ошибок, навигация по метаданным. В комментариях люди пишут с интонацией человека, который двадцать лет шил без напёрстка и наконец получил напёрсток.

YAxUnit — фреймворк для юнит-тестирования в 1С — набрал 27 комментариев, и это не «мы уже используем, давайте обсудим нюансы». Это «ого, так можно?». Статья о тестировании на масштабах с генерацией данных — ещё 27. CI/CD для 1С через EDT — только 11: пишут те, кто уже это делает, и их не очень много. Картина складывается сама.

Я не хочу сказать, что нейросети в 1С-разработке — это пустышка. Qwen Code CLI с BSL Language Server — это реально работающий инструмент, закрывающий реальную боль. Тот же ReAct-агент для аналитических запросов — это, возможно, ближайший путь к нормальному BI поверх 1С:ЗУП для компаний, у которых нет ресурсов на отдельную аналитическую платформу. Это практично. Но мотивация неочевидна снаружи.

Посмотрите на OneBase — проект, получивший 158 комментариев. Человек взял и написал альтернативную платформу: YAML-конфигурации, PostgreSQL, DSL, двадцать мегабайт бинарника. Сообщество пришло в огромном количестве, но встретило проект скептически — не зрелый, не для enterprise, учебная история. Однако сам факт: человек потратил месяцы, чтобы написать это, и 158 человек пришли обсудить — говорит о чём-то живом. Ощущение «платформа нас ограничивает» не просто живёт в сообществе, оно периодически материализуется в конкретные проекты.

Параллельно с нейросетевым хайпом тихо накапливаются сигналы другого рода. Статья о проблеме с типом binarydata в версии 8.3.27 — 31 комментарий. Баг платформы, который задел достаточно людей, чтобы получить такой отклик. Изменения в версии 8.3.26 — 16 комментариев, спокойное профессиональное чтение changelog. Это другой ритм, другой регистр — не хайп, а сопровождение системы.

Если сложить всё вместе, получается неожиданный портрет сообщества. Одни учат Docker в 2026 году. Другие разбираются, какая квантизация у Qwen даёт лучший баланс скорости и качества на RTX 5060ti. Третьи методично разбираются с багами платформы и пишут CI/CD-пайплайны. Все три группы сосуществуют в одном пространстве, и именно это сосуществование — главное, что я вижу в данных за последние тридцать дней.

Нейросетевой ажиотаж рано или поздно схлынет, останутся те, кто нашёл конкретную боль и конкретное решение. Qwen Code CLI с BSL Language Server, скорее всего, останется — потому что закрывает что-то реальное. LangGraph-агенты для аналитики — зависит от того, насколько удобно это окажется в эксплуатации.

А вот вопрос, который мне не даёт покоя: мы обсуждаем, какую нейросеть использовать для генерации 1С-кода, — но кто сейчас работает над самим инструментарием? Нормальным рефакторингом, нормальным статическим анализом, нормальной поддержкой тестов в самой платформе? Или мы негласно согласились, что 1С этого не даст, и просто строим обходные пути поверх?
